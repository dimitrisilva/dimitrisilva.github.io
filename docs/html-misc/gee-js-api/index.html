Introduction

Google Earth Engine (GEE) is a cloud-based platform designed for large-scale geospatial analysis. By combining a vast catalog of geospatial datasets with a distributed computing infrastructure, Earth Engine allows users to process satellite imagery, climate records, elevation models, land-cover products, and many other forms of geographic data without downloading them to a local machine.

One of the main strengths of Earth Engine is that it allows users to focus on analysis rather than data management. Instead of worrying about storage, preprocessing, or computational limitations, users can write concise scripts that execute on Google's servers and take advantage of datasets that would otherwise require substantial computational resources to handle.

This guide introduces the Google Earth Engine JavaScript API from a practical perspective. Rather than attempting to document every available class and method, it focuses on the concepts, object types, workflows, and commands that are most commonly used when developing Earth Engine projects.

The guide is organized around the way Earth Engine projects are typically built. It begins with an introduction to the platform itself, followed by an explanation of Earth Engine's object model, common workflows, visualization techniques, user-interface components, and export mechanisms. The guide concludes with a complete example illustrating how these concepts come together in a practical application.

Google Earth Engine

What is Google Earth Engine?

Google Earth Engine is a platform for storing, processing, visualizing, and analyzing geospatial data at scale.

The platform was originally developed to support environmental monitoring and Earth observation applications, but it has since been adopted in many fields, including remote sensing, environmental science, agriculture, forestry, urban studies, hydrology, climate science, public policy, and disaster management.

A defining characteristic of Earth Engine is that computations are executed on Google's infrastructure rather than on the user's computer. This allows analyses involving terabytes or petabytes of data to be performed using relatively simple scripts.

Earth Engine combines three major components:

  1. A catalog containing thousands of geospatial datasets.
  2. A cloud-computing infrastructure for large-scale processing.
  3. Programming interfaces that allow users to interact with data and computational resources.

Most users interact with Earth Engine through either the JavaScript API or the Python API. This guide focuses on the JavaScript API.

Accessing Earth Engine

Google Earth Engine can be accessed through several interfaces, each designed for a different type of workflow.

The easiest way to begin using Earth Engine is through the Earth Engine Code Editor, a browser-based integrated development environment (IDE) provided by Google. After creating a Google account and registering for Earth Engine, users can access the Code Editor directly from a web browser without installing any software. The Code Editor allows users to write JavaScript code, visualize results on an interactive map, inspect datasets, manage assets, and launch export tasks.

For users who prefer working locally, Earth Engine also provides client libraries for Python and JavaScript. The Python API is commonly used in Jupyter notebooks, Google Colab notebooks, and automated workflows that integrate Earth Engine with data-science libraries such as NumPy, pandas, and GeoPandas. The JavaScript API can be used outside the Code Editor in custom web applications.

A typical path for new users is:

  1. Create a Google account.
  2. Register for Earth Engine access.
  3. Open the Earth Engine Code Editor.
  4. Explore public datasets and run example scripts.
  5. Gradually move to Python notebooks or custom applications when more advanced workflows are required.

For most readers of this guide, the Earth Engine Code Editor is the recommended starting point because it provides immediate access to the full functionality of the platform with minimal setup.

Earth Engine Code Editor

The Earth Engine Code Editor is a web-based integrated development environment specifically designed for the JavaScript API. It allows users to write scripts, execute analyses, visualize outputs, inspect geographic features, manage assets, and export results.

For many users, the Code Editor serves as the primary environment for Earth Engine development and experimentation.

Python API

Earth Engine also provides a Python API that can be used from environments such as Jupyter Notebook, Google Colab, and Visual Studio Code. The Python API is particularly useful when Earth Engine must be integrated with data science, machine learning, or statistical workflows.

Earth Engine Applications

Users can also publish interactive web applications based on Earth Engine analyses. These applications allow non-programmers to explore maps, visualizations, and results through a graphical interface.

The Earth Engine Code Editor

The Code Editor consists of several interconnected components that facilitate the development and execution of Earth Engine scripts.

Script Editor

The script editor is where JavaScript code is written, edited, and executed. It supports syntax highlighting, code organization, and script management.

Map Panel

The map panel displays geographic layers added through commands such as Map.addLayer(). Most visual outputs generated during an analysis appear in this panel.

Console

The console displays printed objects, messages, debugging information, and the results of exploratory analyses.

Inspector

The inspector allows users to click on map locations and examine underlying pixel values, feature attributes, and other information associated with geographic objects.

Tasks Panel

The tasks panel displays export operations initiated through Earth Engine. Users can monitor the progress of exports and manage pending tasks.

Assets Panel

The assets panel provides access to datasets and resources managed by the user, including uploaded files and custom collections.

Main Interface Components

A typical Earth Engine workflow involves constant interaction among the editor, map, console, and tasks panel.

For example, a user may load a satellite dataset, filter it to a specific region and time period, visualize the result on the map, inspect individual pixel values, and ultimately export the final product for further analysis.

Understanding how these interface components interact helps streamline the development process and facilitates debugging.

Earth Engine Workflow

Most Earth Engine projects follow a broadly similar workflow. While the specific datasets and analyses vary, the overall process usually consists of loading data, filtering observations, performing transformations, generating derived products, summarizing results, visualizing outputs, and exporting final datasets.

Loading Data

The first step is typically to load raster or vector datasets from the Earth Engine catalog or from user-managed assets. 


            var s2 = ee.ImageCollection('COPERNICUS/S2_SR');

Filtering Data

Datasets are often filtered according to time, location, cloud coverage, or attribute values. 


            var filtered = s2.filterDate('2024-01-01', '2024-12-31')
              .filterBounds(region);

Manipulating Data

Earth Engine provides a large collection of methods for band selection, mathematical transformations, masking, reprojection, aggregation, and feature processing. 


            var ndvi = image.normalizedDifference(['B8', 'B4']);

Creating Derived Products

Many analyses involve generating new datasets from existing ones, such as vegetation indices, terrain products, classifications, change-detection maps, or temporal composites. 


            var medianImage = filtered.median();

Summarizing Data

Statistical summaries can be computed over regions, collections, or individual datasets. 


          var stats =
            image.reduceRegion({
              reducer: ee.Reducer.mean(),
              geometry: region,
              scale: 30
            });

The sections that follow describe how results can be visualized, incorporated into interactive interfaces, and exported for use in other applications.

Server-side and Client-side Objects

One of the most important concepts in Google Earth Engine is the distinction between server-side and client-side objects. Understanding this distinction is essential for writing correct and efficient Earth Engine programs.

Unlike conventional JavaScript applications, Earth Engine does not immediately retrieve datasets and perform calculations within the browser. Instead, most objects represent instructions that are sent to Google's servers, where the actual computation takes place.

Objects such as: 


            ee.Image
            ee.ImageCollection
            ee.Feature
            ee.FeatureCollection
            ee.Geometry
            ee.Number
            ee.Dictionary
            ee.List

are server-side objects. These objects do not contain the underlying data itself. Rather, they describe datasets and operations that will eventually be executed by Earth Engine.

By contrast, ordinary JavaScript objects are client-side objects. Examples include: 


            var city = 'Campinas';
            var years = [2020, 2021, 2022];
            var settings = {scale: 30};

These objects exist only within the browser and are immediately available to JavaScript.

The distinction becomes important because Earth Engine objects cannot be manipulated in the same way as ordinary JavaScript objects. For example, the following code does not download the image itself: 


            var dem = ee.Image('USGS/SRTMGL1_003');

Instead, the variable dem stores a reference to a dataset and a description of how Earth Engine should access it.

When information must be transferred from the server to the client, methods such as evaluate() and getInfo() are used. 


            feature.get('NAME').evaluate(function(name) {print(name);});
Important: Whenever possible, computations should remain on the server. Moving large amounts of data to the client can significantly reduce performance and eliminate many of the advantages provided by Earth Engine's distributed computing infrastructure.

Many difficulties encountered by new Earth Engine users can be traced to confusion between server-side and client-side objects. Developing an intuitive understanding of where data and computations reside is therefore one of the most valuable skills for working effectively with the platform.

The Earth Engine Object Model

Earth Engine organizes geospatial information through a relatively small set of fundamental object types. Most analyses involve creating, filtering, transforming, visualizing, and exporting one or more of these objects.

At a high level, Earth Engine distinguishes between raster data and vector data.

Raster Data
│
├── ee.Image
└── ee.ImageCollection

Vector Data
│
├── ee.Geometry
├── ee.Feature
└── ee.FeatureCollection

Raster objects represent information as a grid of cells or pixels. Examples include satellite imagery, elevation models, land-cover products, and climate datasets.

Vector objects represent information using points, lines, polygons, and associated attributes. Administrative boundaries, roads, watersheds, and census units are common examples.

The following sections describe the object types that form the foundation of most Earth Engine workflows.

ee.Image

An ee.Image is Earth Engine's fundamental raster object. It represents a single raster dataset consisting of one or more bands.

Examples include:

  • A Sentinel-2 image acquired on a specific date;
  • A digital elevation model;
  • A land-cover classification map;
  • A vegetation-index layer.

Constructor


                ee.Image(assetId)

Main Arguments

Argument Description
assetId Identifier of a raster dataset available in Earth Engine.

Other Arguments and Options

Images may also be created from constants, mathematical expressions, or operations involving other images. In practice, however, many workflows begin by loading an existing image from the Earth Engine catalog.

Common Methods

Method Purpose
select() Selects one or more bands.
clip() Restricts an image to a geometry.
addBands() Adds bands to an image.
rename() Renames bands.
expression() Evaluates mathematical expressions.

Example


                var dem = ee.Image('USGS/SRTMGL1_003');

ee.ImageCollection

An ee.ImageCollection is Earth Engine's primary abstraction for handling temporal and multi-scene raster datasets. Rather than representing a single image, it represents an ordered collection of images that can be filtered, transformed, and summarized.

Many Earth observation datasets are distributed as image collections because they contain repeated observations acquired over time.

Constructor


                ee.ImageCollection(assetId)

Main Arguments

Argument Description
assetId Identifier of an image collection.

Common Methods

Method Purpose
filterDate() Filters images by date.
filterBounds() Filters images by location.
filter() Applies custom filtering conditions.
map() Applies a function to each image.
sort() Sorts images.
median() Creates a median composite.

Example


                var collection = ee.ImageCollection('COPERNICUS/S2_SR')
                  .filterDate('2024-01-01', '2024-12-31');

ee.Geometry

An ee.Geometry represents a geometric object such as a point, line, polygon, or collection of geometries. Geometries are frequently used to define study areas, clipping boundaries, spatial filters, and regions over which statistics should be computed.

Common Constructors


                ee.Geometry.Point([lon, lat])
                ee.Geometry.LineString([...])
                ee.Geometry.Polygon([...])

Main Uses

Purpose Example
Define study areas filterBounds()
Clip rasters clip()
Compute statistics reduceRegion()
Create vector objects ee.Feature()

Example


                var region =
                  ee.Geometry.Polygon([
                    [
                      [-47.5, -23.0],
                      [-47.5, -22.5],
                      [-46.8, -22.5],
                      [-46.8, -23.0]
                    ]
                  ]);

ee.Feature

An ee.Feature combines a geometry with a set of attributes. Conceptually, a feature is similar to a row in a geographic database table.

For example, a municipality may be represented by a polygon geometry and attributes such as its name, population, and administrative code.

Constructor


                ee.Feature(geometry, properties)

Main Arguments

Argument Description
geometry The associated geometry.
properties Dictionary of attributes.

Example


                var city = ee.Feature(ee.Geometry.Point([-47.06, -22.90]), {name: 'Campinas'});

ee.FeatureCollection

An ee.FeatureCollection represents a collection of features. It is Earth Engine's primary vector-data container and is widely used for administrative boundaries, road networks, hydrographic systems, and user-uploaded datasets.

Constructor


                ee.FeatureCollection(assetId)

Common Methods

Method Purpose
filter() Applies attribute filters.
filterBounds() Applies spatial filters.
map() Processes each feature.
merge() Combines collections.
size() Counts features.

Example


                var municipalities = ee.FeatureCollection('FAO/GAUL_SIMPLIFIED_500m/2015/level2');

Visualization

Visualization is one of the most important components of an Earth Engine workflow. Although Earth Engine supports sophisticated geospatial analysis, most projects eventually require results to be displayed on a map so that they can be interpreted, validated, and communicated.

Map visualization in Earth Engine is primarily performed through the Map object. Two of the most commonly used methods are Map.addLayer() and Map.centerObject().

Map.addLayer()

The Map.addLayer() method adds a layer to the map display. Layers may represent images, image collections, feature collections, geometries, or other spatial objects.

Syntax


                Map.addLayer(object, visParams, name, shown, opacity);

Main Arguments

Argument Description
object Object to be displayed.
visParams Visualization parameters.
name Layer name shown in the layer manager.
shown Whether the layer is initially visible.
opacity Layer transparency.

Example


                Map.addLayer(
                  ndvi,
                  {min: 0, max: 1, palette: ['white', 'green']},
                  'NDVI'
                );

Map.centerObject()

The Map.centerObject() method automatically centers the map on a specified object and optionally applies a zoom level.

Syntax


                Map.centerObject(object, zoom);

Main Arguments

Argument Description
object Object used as reference.
zoom Optional zoom level.

Example


                Map.centerObject(region, 10);

Visualization Parameters

Visualization parameters control how raster data are rendered on the screen. They do not modify the underlying data; they merely affect its appearance.

Parameter Purpose
min Minimum display value.
max Maximum display value.
palette Color palette.
bands Bands used for visualization.
gamma Gamma correction.

For multiband imagery, visualization often involves selecting three bands corresponding to red, green, and blue channels. 


              Map.addLayer(
                image,
                {bands: ['B4', 'B3', 'B2'], min: 0, max: 3000},
                'True Color'
              );

Building Interactive Interfaces

Earth Engine includes a lightweight user-interface library that can be used to create interactive applications directly within the Code Editor. These interfaces allow users to select datasets, adjust parameters, display text, and interact with maps.

ui.Panel

A ui.Panel is a container that organizes user-interface elements.

Common Uses

  • Sidebars
  • Control panels
  • Information panels
  • Legend containers

Example


                var panel = ui.Panel({style: {width: '300px'}});

ui.Label

A ui.Label displays text within the interface.

Example


                var title = ui.Label('Land Cover Explorer');

Labels are commonly used for titles, descriptions, instructions, and dynamic outputs.

ui.Select

A ui.Select creates a drop-down menu that allows users to choose among multiple options.

Main Arguments

Argument Description
items Available options.
value Default value.
onChange Function executed after selection.

Example


                var selector =
                  ui.Select({
                    items: [
                      'NDVI',
                      'Elevation',
                      'Land Cover'
                    ]
                  });

When combined with panels, labels, maps, and event handlers, selection widgets can be used to build sophisticated interactive applications.

Exporting Results

Although Earth Engine provides extensive visualization and analysis capabilities, many workflows ultimately require results to be exported for use in GIS software, statistical environments, machine learning pipelines, or web applications.

Export operations are submitted as tasks that appear in the Tasks panel of the Code Editor.

Export.image.toDrive()

Exports a raster image to Google Drive. 


              Export.image.toDrive({
                image: image,
                description: 'output',
                scale: 30,
                region: region
              });

Export.table.toDrive()

Exports vector data or tabular results to Google Drive. 


              Export.table.toDrive({
                collection: features,
                description: 'features'
              });

Export.image.toAsset()

Exports a raster image to the user's Earth Engine assets. 


              Export.image.toAsset({
                image: image,
                description: 'assetExport',
                assetId: 'users/user/output'
              });
Note: Export operations are not executed immediately. After a task is created, it must usually be started manually from the Tasks panel.

              // Global Administrative Unit Layers - Vector datasets 
              var municipalities = ee.FeatureCollection('FAO/GAUL/2015/level2')
                .filter(ee.Filter.and(
                  ee.Filter.eq('ADM0_NAME', 'Brazil'), 
                  ee.Filter.eq('ADM1_NAME', 'Sao Paulo')
              ));

              // Municipalities of Campinas mesoregion
              var campinasMicroregions = {
                'Campinas': [
                  'Campinas', 'Americana', 'Cosmopolis', 'Elias Fausto', 'Holambra', 
                  'Hortolandia', 'Indaiatuba', 'Itatiba', 'Jaguariuna', 'Monte Mor', 
                  'Morungaba', 'Nova Odessa', 'Paulinia', 'Santo Antonio De Posse', 
                  'Sumare', 'Valinhos', 'Vinhedo'
                ],
                'Amparo': [

                  'Pedreira', 'Serra Negra', 'Socorro'
                ],
                'Moji-mirim': [
                  'Moji-mirim', 'Artur Nogueira', 'Conchal', 'Engenheiro Coelho',
                  'Estiva Gerbi', 'Itapira', 'Moji-guacu'
                ],
                'Pirassununga': [
                  'Pirassununga', 'Aguai', 'Analandia', 'Descalvado',
                  'Santa Cruz Da Conceicao', 'Santa Cruz Das Palmeiras', 'Tambau'
                ],
                'Sao Joao Da Boa Vista': [
                  'Sao Joao Da Boa Vista', 'Aguas Da Prata', 'Caconde', 'Casa Branca', 
                  'Divinolandia', 'Espirito Santo Do Pinhal', 'Itobi', 'Mococa', 
                  'Santo Antonio Do Jardim', 'Sao Jose Do Rio Pardo', 
                  'Sao Sebastiao Da Grama', 'Vargem Grande Do Sul'
                ]
              };

              // All municipality names across all microregions of Campinas mesoregion
              var allMuns = Object.keys(campinasMicroregions).reduce(
                function(acc, key) {return acc.concat(campinasMicroregions[key]);}, []
              );

              // Filter municipalities in vector data
              var campinasMesoregion = municipalities.filter(
                ee.Filter.inList('ADM2_NAME', allMuns)
              );

              print(campinasMesoregion.aggregate_array('ADM2_NAME').sort());

              // Build mesoregion boundaries by dissolving municipality boundaries
              var mesoregion = municipalities
                .filter(ee.Filter.inList('ADM2_NAME', allMuns))
                .union(1)
                .set('mesoregion', 'Campinas');

              // Build microregion boundaries by dissolving municipality boundaries
              var microregionFeatures = Object.keys(campinasMicroregions).map(function(key) {
                var names = campinasMicroregions[key];
                var dissolved = municipalities
                  .filter(ee.Filter.inList('ADM2_NAME', names))
                  .union(1) // ee.FeatureCollection.union(maxError) -> FeatureCollection
                  .first() // FeatureCollection to Feature
                  .set('microregion', key); // Adding property 'microregion' for feature
                return dissolved;

                .filter(ee.Filter.inList('ADM2_NAME', Object.keys(campinasMicroregions)));

              var mesoGeom = mesoregion.geometry();

              var dem = ee.Image('USGS/SRTMGL1_003').clip(mesoGeom);

              var hillshade = ee.Terrain.hillshade(dem); // Hillshade derived from DEM

              // Styled features
              var munFeatures = campinasMesoregion.style(

              );

              var mainFeatures = mainMuns.style(
                {lineType: 'solid', color: 'ff0000ff', fillColor: 'ff000077', width: 1.00}
              );

              var microFeatures = microregions.style(
                {lineType: 'solid', color: '000000ff', fillColor: 'd3d3d300', width: 1.25}
              );

              var mesoFeature = mesoregion.style(
                {lineType: 'solid', color: '000000ff', fillColor: 'd3d3d300', width: 1.25}
              );

              var demVis = {
                min: 500,
                max: 1800,
                palette: [
                  '0b3d0b', // dark green (lowest)
                  '2e8b57', // green
                  '7fbf7f', // light green
                  'd9c27f', // tan
                  'a67c52', // brown
                  'ffffff'  // white (highest)
                ]
              };

              // Converting DEM and hillshade to RGB images
              var elevationRgb = dem.visualize(demVis); // Colored terrain image

              // Center map
              Map.centerObject(campinasMesoregion, 8);

              // Add layers
              Map.addLayer(dem, demVis, 'Elevation');
              //Map.addLayer(hillshade, {min: 0, max: 255}, 'Hillshade', false);

              Map.addLayer(munFeatures, {opacity: 1.00}, 'Municipalities');
              Map.addLayer(mainFeatures, {opacity: 1.00}, 'Main municipalities');
              Map.addLayer(microFeatures, {opacity: 1.00}, 'Microregions of Campinas Mesoregion');
              Map.addLayer(mesoFeature, {opacity: 1.00}, 'Campinas Mesoregion');

              // Geometry information panel
              var municipalityLabel = ui.Label(
                'Click on a municipality', {fontSize: '14px', fontWeight: 'bold'}
              );

              var infoPanel = ui.Panel({
                widgets: [municipalityLabel],
                style: {position: 'bottom-left', padding: '8px'}
              });

              Map.add(infoPanel);

              // Elevation information panel
              var demLegend = ui.Panel({
                style: {
                  position: 'bottom-right',
                  padding: '8px',
                  backgroundColor: 'ffffffcc'
                }
              });

              demLegend.add(ui.Label('Elevation (m)', {fontWeight: 'bold'}));

              function addLegendRow(color, label) {
                var colorBox = ui.Label('', {
                  backgroundColor: '#' + color,
                  padding: '8px',
                  margin: '0 4px 0 0'
                });

                var text = ui.Label(label);

                demLegend.add(ui.Panel([colorBox, text], ui.Panel.Layout.flow('horizontal')));
              }

              addLegendRow('0b3d0b', '500 m');
              addLegendRow('2e8b57', '680 m');
              addLegendRow('7fbf7f', '860 m');
              addLegendRow('d9c27f', '1040 m');
              addLegendRow('a67c52', '1220 m');
              addLegendRow('ffffff', '1400 m');

              Map.add(demLegend);

              // Click handler
              var highlightLayer;

              Map.onClick(function(coords) {

                var point = ee.Geometry.Point([coords.lon, coords.lat]);

                var municipality = campinasMesoregion.filterBounds(point);

                municipality.first().get('ADM2_NAME').evaluate(function(name) {
                  municipalityLabel.setValue(
                    name ? 'Municipality: ' + name : 'No municipality selected'
                  );
                });

                var highlighted = municipality.style(

                if (highlightLayer) {

                }

                highlightLayer = ui.Map.Layer(highlighted, {}, 'Selected municipality');

                Map.layers().add(highlightLayer);

              });

Understanding the Script

The example script follows a typical Earth Engine workflow and can be understood as a sequence of five steps.

Step 1 – Define the Study Area

The script begins by defining the geographic region that will be analysed. This is typically accomplished using an ee.Geometry object or a feature collection containing administrative boundaries. The study area is subsequently used to filter datasets and limit computations to the region of interest.

Step 2 – Load a Dataset

The script then loads one or more datasets from the Earth Engine Data Catalog. Depending on the application, these datasets may consist of satellite imagery (ee.ImageCollection), vector boundaries (ee.FeatureCollection), elevation models, climate data, or other geospatial resources.

Step 3 – Filter and Prepare the Data

The loaded dataset is filtered according to criteria such as date range, geographic extent, cloud cover, or metadata attributes. Additional preprocessing operations may also be applied, including band selection, masking, clipping, or image transformations.

Step 4 – Produce a Derived Result

Once the data have been prepared, the script computes a derived product. Examples include vegetation indices, land-cover classifications, temporal composites, summary statistics, or spatial aggregations. This stage typically involves applying functions to images or collections and generating new Earth Engine objects.

Step 5 – Visualize the Result

Finally, the result is displayed on the interactive map using Map.addLayer(). Visualization parameters determine how values are represented through colours and stretching. The map is then centred on the study area using Map.centerObject(), allowing the user to inspect the output interactively.
