Self-Study Plan for Mastering Google Cloud Platform (GCP) for Data Science and Machine Learning

This syllabus is designed as a progressive self-study plan to build proficiency in GCP, leveraging a background in Python, R, SQL, and econometrics. It emphasizes cloud-native tools for data ingestion, storage, processing, analysis, and machine learning, while minimizing overlap between textbooks. The primary focus is on free-tier GCP services (e.g., BigQuery sandbox, limited Cloud Storage, free tiers of Pub/Sub, Dataflow for small jobs, Vertex AI notebooks with constraints, and Google Earth Engine public datasets). No paid features are required; stick to sandbox limits (e.g., 1 TB/month query in BigQuery) and monitor usage to avoid charges. The plan is divided into modules, each building skills toward specific exercises. Allocate 1-2 weeks per module based on your pace, with hands-on practice integrated.


Module 1: GCP Fundamentals and Setup

This module introduces GCP basics, including account setup, navigation, and core concepts like serverless computing and separation of compute/storage. Relevance: Data scientists need a scalable, collaborative environment to handle large datasets without managing infrastructure, enabling focus on analysis and ML rather than ops.

Material to study.

  • Primary: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 2 (Big Data Capabilities on GCP), focusing on console intro, service mapping, quotas, and serverless concepts.
  • Alternatives: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapter 1 (sections on Cloud Computing and Serverless) for a data-focused perspective.

Exercises.

  • Set up a free GCP account and explore the console: Pin services like BigQuery, Cloud Storage, and Pub/Sub; create a project and check quotas via Cloud Shell (gcloud commands).
  • Toward Exercise 2 (Brazilian bonds): Use Cloud Shell to download sample CSV bond data from a public site (e.g., via wget to a public dataset proxy) and upload to Cloud Storage.
  • Toward Exercise 3 (Google Trends): Query public Google Trends data in BigQuery's public datasets (e.g., search for trends samples) and preview in console.
Module 2: Data Storage and Warehousing with BigQuery

Learn BigQuery's architecture, data loading, schema design, and basic SQL queries. Relevance: BigQuery is a serverless data warehouse for petabyte-scale analytics; it's essential for economists/data scientists to store, query, and analyze structured data efficiently for time series, trends, or geospatial insights.

Material to study.

  • Primary: Google BigQuery: The Definitive Guide (Lakshmanan and Tigani, 2019) - Chapters 1 (What Is Google BigQuery?), 2 (Query Essentials), and 4 (Loading Data into BigQuery).
  • Alternatives: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 3 (Building a Data Warehouse in BigQuery) for practical loading examples.

Exercises.

  • Create a BigQuery dataset in sandbox mode, load a local CSV (e.g., sample news or bond data), and run basic queries (SELECT, WHERE, GROUP BY).
  • Toward Exercise 1 (YouTube news): Load sample JSON from YouTube API responses into BigQuery, query to extract titles/points.
  • Toward Exercise 2 (Brazilian bonds): Load daily bond prices CSV into a partitioned table (by date), compute summary stats with SQL (e.g., AVG, STDDEV).
  • Toward Exercise 4 (Earth Engine): Query public geospatial datasets in BigQuery (e.g., NOAA weather) and visualize simple aggregates.
Module 3: Advanced Querying and Optimization in BigQuery

Cover data types, functions, joins, arrays/structs, partitioning/clustering, and performance optimization. Relevance: Efficient querying reduces costs and enables complex econometric analyses like time series modeling or statistical summaries on large datasets.

Material to study.

  • Primary: Google BigQuery: The Definitive Guide (Lakshmanan and Tigani, 2019) - Chapters 3 (Data Types, Functions, and Operators) and 7 (Optimizing Performance and Cost).
  • Alternatives: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapter 3 (sections on Schema Exploration and Querying Using BigQuery).

Exercises.

  • Use TIMESTAMP functions to handle dates in a table, partition by date, and optimize a join query.
  • Toward Exercise 2 (Brazilian bonds): Query bond data for time series stats (e.g., rolling averages with WINDOW functions), optimize for low scan size.
  • Toward Exercise 3 (Google Trends): Join Trends data with external sources (federated query if free), compute correlations or aggregates.
  • Toward Exercise 1 (YouTube news): Use STRING functions to parse and summarize text fields from loaded YouTube data.
Module 4: Data Ingestion and Batch Processing

Ingest data from various sources, use Cloud Storage for staging, and process batches with Dataflow or Spark on Dataproc (free-tier limits). Relevance: Automates data pipelines for real-world DS/ML, handling ETL/ELT for econometric data or trends.

Material to study.

  • Primary: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapter 2 (Ingesting Data into the Cloud).
  • Alternatives: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 5 (Building a Data Processing Pipeline) for Dataproc examples; Google BigQuery Guide - Chapter 4 (federated queries section).

Exercises.

  • Ingest CSV/JSON to Cloud Storage, load to BigQuery via Dataflow template (small job).
  • Toward Exercise 2 (Brazilian bonds): Script in Python (Cloud Shell) to download bond data daily, stage in Storage, load to BigQuery.
  • Toward Exercise 4 (Earth Engine): Export small Earth Engine dataset to Storage, ingest to BigQuery for stats.
Module 5: Streaming Data Processing

Handle real-time data with Pub/Sub and Dataflow for streaming pipelines. Relevance: Enables near-real-time analysis for dynamic data like news or trends, crucial for time-sensitive econometric models.

Material to study.

  • Primary: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapter 4 (Streaming Data: Publication and Ingest).
  • Alternatives: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 6 (Building a Data Processing Pipeline with Streaming Data); Streaming Systems (Akidau et al., 2018) - Chapters on windowing/time (theoretical complement).

Exercises.

  • Create Pub/Sub topic, publish messages, process stream with Dataflow (windowing/aggregation).
  • Toward Exercise 1 (YouTube news): Simulate streaming YouTube links via Pub/Sub, process to extract/summarize points.
  • Toward Exercise 3 (Google Trends): Stream mock Trends updates, aggregate in real-time.
Module 6: Interactive Exploration and Visualization

Use Vertex AI notebooks for EDA, build dashboards with Looker Studio. Relevance: Facilitates exploratory analysis and visualization of models/results, key for interpreting econometric outputs or geospatial data.

Material to study.

  • Primary: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapters 3 (Creating Compelling Dashboards) and 5 (Interactive Data Exploration).
  • Alternatives: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 7 (Visualizing Data with Looker Studio).

Exercises.

  • In Vertex AI notebook (free instance), query BigQuery with pandas/R, plot distributions.
  • Toward Exercise 2 (Brazilian bonds): EDA in notebook: plot time series, compute stats.
  • Toward Exercise 3 (Google Trends): Build Looker dashboard for statistical visuals.
  • Toward Exercise 1 (YouTube news): Visualize summarized news points in dashboard.
Module 7: Machine Learning on GCP

Build ML models with BigQuery ML and Vertex AI (free tiers). Relevance: Integrates SQL-based ML for forecasting/time series, bridging your econometric knowledge to scalable ML.

Material to study.

  • Primary: Google BigQuery: The Definitive Guide (Lakshmanan and Tigani, 2019) - Chapter 9 (Machine Learning in BigQuery).
  • Alternatives: Data Science on the Google Cloud Platform (Lakshmanan, 2022) - Chapters 6 (Bayesian Classifier) and 7 (Statistical Time-Series Forecasting); Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapter 8 (ML on GCP).

Exercises.

  • Train simple regression in BigQuery ML on sample data.
  • Toward Exercise 2 (Brazilian bonds): Forecast bond prices with time-series model in BigQuery ML.
  • Toward Exercise 4 (Earth Engine): Simple ML on geospatial data (e.g., classify land cover).
Module 8: Advanced Topics: Geospatial, Orchestration, and Integration

Orchestrate pipelines with Composer, handle GIS, integrate APIs. Relevance: Automates end-to-end workflows for complex projects like news summarization or geospatial stats.

Material to study.

  • Primary: Data Engineering with Google Cloud Platform (Wijaya, 2024) - Chapters 4 (Building Workflows with Cloud Composer) and 9 (Geospatial in BigQuery).
  • Alternatives: Google BigQuery Guide - Chapter 10 (GIS); Data Science on GCP - Chapter 9 (ML with TensorFlow).

Exercises.

  • Create DAG in Composer to orchestrate ingestion/query.
  • Toward Exercise 4 (Earth Engine): Analyze geospatial data (e.g., climate trends) with GIS functions.
  • Toward Exercise 1 (YouTube news): Orchestrate pipeline for link ingestion, summary (use free Vertex AI for translation if available), audio output via Python.
Module 9: Capstone Integration and Projects

Synthesize skills into full pipelines. Relevance: Applies GCP to real DS/ML workflows, closing knowledge gaps.

Material to study.

  • Review all books' summaries; Streaming Systems for advanced streaming if needed.

Exercises. Complete full projects:

  • Exercise 1: Pipeline for YouTube news (ingest links via Pub/Sub, process in Dataflow, summarize in notebook, output audio).
  • Exercise 2: Bond analysis pipeline (ingest, model in BigQuery ML, dashboard).
  • Exercise 3: Trends stats in BigQuery/Looker.
  • Exercise 4: Earth Engine geospatial stats (e.g., deforestation analysis, visualize).
